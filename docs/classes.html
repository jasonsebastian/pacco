Classes¶

This module shall contain the representation of object PackageRegistry represent a package collection and client pair. BinaryPackage represent a binary based on the setting in a PackageRegistry. BinaryPackage shall not know which PackageRegistry it is in. To make this happen, we change the working directory of client to be inside the PackageRegistry.

The dependency graph of the modules related composed of PackageRegistry ---> Client BinaryRegistry ---> Client PackageRegistry ---> BinaryRegistry

class pacco.classes.PackageRegistry(name, client, allowed_settings=None)¶

An instantiation of this class represents the existence of the package collection self.name in the client self.client.

There is two way to instantiate the PackageRegistry, with or without allowed_settings If it is with allowed_settings, it will take the allowed_settings as the setting, and will try to set up a new package if it’s not set up, or if the package already exist, will check if the settings is equal else it will raise warning and use the existing allowed_settings version.

Thus, only use the allowed_settings options when creating new package registry.
